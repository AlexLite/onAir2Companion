# companion-module-onair-bridge

Companion v4 module for the custom onAir bridge server.

## Build

```bash
npm install
npm run build
```

## Local module in Companion v4

1. Open Companion settings.
2. Set **Developer modules path** to the parent folder containing this module directory.
3. Restart Companion.
4. Add connection **onair-bridge**.

## Connection settings

- Host: `192.168.11.71`
- Port: `27015`

## Available actions

- Send command (`action=send` JSON request to bridge)
- Send template command (FDOnAir catalog)
- Send JSON line (raw JSONL)
- Health check (`{"action":"health"}`)

## Presets

- PLAY
- STOP
- NEXT
- TAKE
- PING
- HEALTH
- FDOnAir template presets (auto-generated by categories):
- `FDOnAir Player`
- `FDOnAir Shedule`
- `FDOnAir Folder`
- `FDOnAir Setting`
- `FDOnAir Loading`
- `FDOnAir MirrorSave` (N+1)

Preset commands are configurable in module config:

- Preset PLAY command
- Preset STOP command
- Preset NEXT command
- Preset TAKE command

## Incoming events handling

The module parses JSON lines from bridge and updates variables:

- `bridge_state`
- `last_message`
- `last_body`
- `last_subject`
- `last_from_machine`
- `last_from_queue`
- `last_response`
- `last_error`

Use feedback `last_body_contains` to react on onAir messages.
Use feedback `message_match` for templates:

- contains
- equals
- regex

Optional filters are available for `subject` and `from_queue`.
